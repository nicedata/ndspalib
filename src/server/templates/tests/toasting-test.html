Toasts via server side events

Synopsis

When a button is clicked, a request to url /toast is sent.
The server responds with an x-nd-event in the response header along with a time stamp string (like "Last toast stamp : 14.02.2026 13:52:11") in the response.
If a redirect_url sits in the toast response, a redirect is tried.

The x-nd-event detail is transformed into a toast message, and the response body is used to update the target content.

Usage

<button id="b1" class="btn btn-primary w-100" nd-link nd-url="/toast" nd-target="#target">Toast button</button>
